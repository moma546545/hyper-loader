### FREEZE-01 🔴 [Critical] — `tools_view._apply_bulk_rename()` Blocks Main Thread with `os.walk`
**File:** `ui/views/tools_view.py` lines 8718–8742

**Diagnosis:** `os.walk()` and `os.rename()` run synchronously on the main GUI thread. On a folder with thousands of files or a network share, this completely freezes the UI.

**Refactored Code:**
```python
# In tools_view.py — add:
from PySide6.QtCore import QThread, Signal as _Sig

class _BulkRenameWorker(QThread):
    progress = _Sig(int, int)   # (done, total)
    finished = _Sig(int)         # renamed_count
    error = _Sig(str)

    def __init__(self, folder: str, template: str, parent=None):
        super().__init__(parent)
        self._folder = folder
        self._template = template

    def run(self):
        from core.smart_rename import build_filename
        count = 0
        files = [
            (root, name)
            for root, _, names in os.walk(self._folder)
            for name in names
        ]
        total = len(files)
        for idx, (root, name) in enumerate(files):
            if self.isInterruptionRequested():
                break
            full = os.path.join(root, name)
            base, ext = os.path.splitext(name)
            if not ext:
                self.progress.emit(idx + 1, total)
                continue
            new_rel = build_filename(self._template, title=base, ext=ext.lstrip("."), channel="", quality="")
            new_full = os.path.join(root, new_rel)
            if os.path.abspath(full) != os.path.abspath(new_full):
                os.makedirs(os.path.dirname(new_full), exist_ok=True)
                try:
                    os.rename(full, new_full)
                    count += 1
                except Exception as exc:
                    logger.warning(f"Bulk rename failed: {exc}")
            self.progress.emit(idx + 1, total)
        self.finished.emit(count)

# In ToolsView._apply_bulk_rename():
def _apply_bulk_rename(self):
    folder = self.rename_folder_input.text().strip()
    if not folder or not os.path.isdir(folder):
        ErrorHandler.show_warning(self, _("Error"), _("Please select a valid folder."))
        return
    template = self.rename_template_combo.currentText().strip()
    self._rename_worker = _BulkRenameWorker(folder, template, self)
    self._rename_worker.finished.connect(
        lambda n: ErrorHandler.show_info(self, _("Done"), _(f"Renamed {n} files."))
    )
    self._rename_worker.finished.connect(lambda _: setattr(self, "_rename_worker", None))
    self._rename_worker.start()
```

### FREEZE-02 🟠 [High] — `main.py` Headless Batch: `app.exec()` Called Multiple Times on Same Loop
**File:** `main.py` lines 158–182, called from 196–204

**Diagnosis:** `_run_headless_batch()` calls `_run_headless_download()` for each URL. Each call reaches `app.exec()` which blocks until `QTimer.singleShot(0, app.quit)` fires. After the first `exec()` returns (app quit), the second URL calls `exec()` again on the same `QCoreApplication` — this is technically valid in Qt but the DownloadWorker from the previous iteration is not waited on before the next starts. If the previous worker emits `DownloadFinishedEvent` *after* the next `exec()` starts, the wrong `_on_finished` handler fires because both are subscribed to `event_bus`.

**Refactored Code:**
```python
def _run_headless_batch(args: argparse.Namespace, urls: list[str]) -> int:
    """Process all URLs through a single shared QCoreApplication event loop."""
    _ensure_headless_logging()
    app = QCoreApplication.instance() or QCoreApplication([sys.argv[0]])
    results: list[dict] = []
    failures = 0
    total = len(urls)
    pending: list[str] = list(urls)

    def _process_next():
        if not pending:
            app.quit()
            return
        index = total - len(pending)
        url = pending.pop(0)
        print(f"[headless] ({index + 1}/{total}) starting: {url}", flush=True)
        item_args = argparse.Namespace(**vars(args))
        item_args.url = url
        worker = _build_headless_worker(item_args)
        _wid = _HEADLESS_WORKER_ID + f"_{index}"
        worker.worker_id = _wid

        if hasattr(worker, "log"):
            worker.log.connect(_print_worker_log)

        def _on_done(event: DownloadFinishedEvent):
            if getattr(event, "worker_id", None) != _wid:
                return
            event_bus.unsubscribe(DownloadFinishedEvent, _on_done)
            ok = bool(event.success)
            results.append({"index": index + 1, "url": url, "success": ok})
            if not ok:
                nonlocal failures
                failures += 1
                if bool(getattr(args, "fail_fast", False)):
                    print("[headless] fail-fast triggered.", file=sys.stderr, flush=True)
                    app.quit()
                    return
            QTimer.singleShot(0, _process_next)  # schedule next without re-entering loop

        event_bus.subscribe(DownloadFinishedEvent, _on_done)
        worker.start()

    QTimer.singleShot(0, _process_next)
    app.exec()  # single shared event loop for all URLs
    print(f"[headless] completed: total={total}, failures={failures}", flush=True)
    return 0 if failures == 0 else 1
```

### FREEZE-04 🟡 [Medium] — `TrimDialog._init_preview()` QNetworkReply Not Retained — Potential Use-After-Free
**File:** `ui/trim_dialog.py` lines 4148–4157

**Diagnosis:**
```python
reply = self.net_manager.get(request)
reply.finished.connect(lambda r=reply: self._on_thumb_loaded(r))
# ← reply is local, no self._reply = reply
```
If `TrimDialog` is closed before the network reply completes, the C++ reply object may be deleted by Qt's memory management. The lambda holds a Python reference to the wrapper, but the underlying C++ object can still be invalid, causing a crash in `reply.readAll()`.

**The Fix:**
```python
# In TrimDialog._init_preview():
self._pending_reply = self.net_manager.get(request)
self._pending_reply.finished.connect(self._on_thumb_loaded_safe)

def _on_thumb_loaded_safe(self):
    reply = self._pending_reply
    self._pending_reply = None
    if reply is None:
        return
    try:
        self._on_thumb_loaded(reply)
    except RuntimeError:
        pass  # C++ object already deleted
    finally:
        reply.deleteLater()

def closeEvent(self, event):
    if hasattr(self, "_pending_reply") and self._pending_reply is not None:
        try:
            self._pending_reply.abort()
            self._pending_reply.deleteLater()
        except RuntimeError:
            pass
        self._pending_reply = None
    super().closeEvent(event)
```

### ARCH-01 — `app.py`'s `_update_speed_history()` — Business Logic in the View Layer
**Target:** `app.py` line 1166–1194

**Issue:** This is the only method in `app.py` with real business logic (50 lines with a `deque`, eviction policy, active worker inspection). Every other method is a one-liner delegate. This method belongs in `DownloadController`, not the main window.

**Solution:**
```python
# Move entirely to core/window_controllers/download_controller.py:

class DownloadController:
    # In __init__:
    self._speed_history: dict[int, deque] = {}
    self._speed_history_max_workers: int = 512

    def update_speed_history(self, wid, kbps: float) -> None:
        try:
            key = int(wid)
        except (TypeError, ValueError, OverflowError):
            return
        history = self._speed_history.setdefault(key, deque(maxlen=60))
        history.append(max(0.0, float(kbps or 0.0)))
        self._evict_stale_speed_history(key)

    def _evict_stale_speed_history(self, protected_key: int) -> None:
        max_workers = self._speed_history_max_workers
        if len(self._speed_history) <= max_workers:
            return
        with self.window._active_workers_lock:
            active_ids = set(self.window.active_workers.keys())
        protected = active_ids | {protected_key}
        display_wid = getattr(self.window, "_display_progress_wid", None)
        if display_wid is not None:
            protected.add(display_wid)
        stale = [k for k in self._speed_history if k not in protected]
        overflow = len(self._speed_history) - max_workers
        for k in stale[:max(0, overflow)]:
            self._speed_history.pop(k, None)

    def get_speed_history(self, wid) -> deque:
        return self._speed_history.get(int(wid), deque(maxlen=60))

# In app.py — replace 50-line method with:
def _update_speed_history(self, wid, kbps):
    self.download_controller.update_speed_history(wid, kbps)
```

### ARCH-02 — `wire_views()` Growing to 70+ Lines — Needs a Signal Registry Pattern
**Target:** `ui/layout_manager.py` lines 1022–1088

**Issue:** `wire_views()` is a 70-line procedural function that hardcodes every signal-slot pair. Adding a new view requires editing this function. If a view fails to initialize, all subsequent wires below it also fail silently.

**Solution:**
```python
# Replace with a data-driven registry:
_VIEW_WIRES: list[tuple[str, str, str, str]] = [
    # (source_attr, signal_name, target_attr, slot_name)
    ("search_view", "analyze_requested",      None,             "_start_analyze"),
    ("search_view", "paste_requested",         None,             "_paste_clipboard"),
    ("search_view", "download_requested",      None,             "_start_single_download"),
    ("search_view", "schedule_requested",      None,             "_add_current_to_queue"),
    ("search_view", "clear_history_requested", None,             "_clear_search_history"),
    ("search_view", "trim_toggle_requested",   None,             "_toggle_trim_options"),
    ("downloads_view", "filter_changed",       None,             "_set_downloads_filter"),
    # ... etc
]

def wire_views(window):
    for src_attr, sig_name, tgt_attr, slot_name in _VIEW_WIRES:
        source = getattr(window, src_attr, None)
        target = getattr(window, tgt_attr, None) if tgt_attr else window
        if source is None or target is None:
            logger.warning(f"wire_views: skipping {src_attr}.{sig_name} → {slot_name} (source or target missing)")
            continue
        sig = getattr(source, sig_name, None)
        slot = getattr(target, slot_name, None)
        if sig is None or slot is None:
            logger.warning(f"wire_views: {src_attr}.{sig_name} or {slot_name} not found")
            continue
        try:
            sig.connect(slot)
        except Exception as exc:
            logger.error(f"wire_views: failed {src_attr}.{sig_name} → {slot_name}: {exc}")
```

### ARCH-03 — `main.py` Headless: `event_bus` Subscription Leak on Exception
**Target:** `main.py` lines 166–192

**Issue:**
```python
event_bus.subscribe(DownloadFinishedEvent, _on_finished)
try:
    worker.start()
    app.exec()
    ...
finally:
    event_bus.unsubscribe(DownloadFinishedEvent, _on_finished)
```
This pattern is correct — the `finally` ensures cleanup. But if `_on_finished` is never called (because the worker raises an exception before emitting), `result.completed` stays `False` and the function returns exit code `1` correctly. However, if `app.exec()` times out or gets interrupted via `KeyboardInterrupt`, the `finally` block runs but `worker.wait(2000)` may keep the process alive 2 extra seconds. Use `worker.quit()` before `wait()`:

**The Fix:**
```python
finally:
    event_bus.unsubscribe(DownloadFinishedEvent, _on_finished)
    try:
        if worker.isRunning():
            worker.requestInterruption()
            worker.quit()         # ← send quit to the QThread event loop first
            worker.wait(1000)     # ← shorter timeout
    except Exception:
        pass
```
